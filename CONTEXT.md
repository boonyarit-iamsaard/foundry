# boonyarit.me

The personal portfolio brand of Boonyarit Iamsa-ard. This glossary fixes the
language used when talking about how the site presents skills, stack, and
engineering identity, so copy and components stay aligned with the brand.

## Language

**Online representation (site purpose)**:
What boonyarit.me is for: a credible home base that puts Boonyarit's engineering
judgment on display — through the work, the writing, and how he frames himself —
so a visitor forms an accurate, high-trust picture of how he thinks and can reach
him for whatever fits (a role _or_ serious work). The door is open both ways.
_Avoid_: portfolio-as-funnel, lead-gen, hire-me page

**Demonstrates, doesn't solicit (the boundary)**:
The site shows judgment and leaves an open invitation; it never hustles. No
"freelance available" hard-sell, and equally no desperate "please hire me." The
craftsman-founder direction is visible in the _character_ of the work, neither
hidden nor advertised — a reader looking for it will see it. Quiet authority,
not a pitch.

**Engineering mind**:
What Boonyarit wants judged: how he thinks — system design, domain knowledge,
handling real complexity — as distinct from familiarity with any given tool.
The site's claim is "strength is how I think, not the tool."
_Avoid_: skills, talent, expertise

**The earned constraint**:
Reliability over cleverness, traced to 19 years in aircraft maintenance where a
careless fix grounded a plane. It is the brand's bridge from the unconventional
story (the hook) to the engineering philosophy (the proof) — not a values-poster
adjective but a lived constraint.
_Avoid_: passion for quality, attention to detail

**Serious internals, simple externals**:
Boonyarit's guiding instinct, used verbatim: explicit structure, clear
boundaries, transactional integrity inside; a calm, simple surface outside.
Complexity is admitted only when it earns its place.

**Primary stack**:
TypeScript / Node.js / React / Next.js — where Boonyarit has the deepest
intuition.
_Avoid_: main tech, core stack (as a categorized resume label)

**Second stack**:
Spring Boot / Java — his active second ecosystem via a year-long client
engagement inside a large insurer's microservices codebase, and the house
language of the fintech/enterprise work he targets in Thailand. Present, not
buried.

**The insurance engagement**:
The confidential client engagement that backs the second stack. The client
(AIA) is never named; the _domain_ — enterprise insurance, real business
complexity, a microservices platform — is expressed, because it is precisely
the work Boonyarit targets. Phrase as "a large insurer" / "enterprise insurance
systems," never the brand.
_Avoid_: naming the client, logos, project specifics under NDA

**Honesty guardrail (stack)**:
What the copy may and may not claim. SQL is the data strength — PostgreSQL and
Azure SQL Server; NoSQL is "rarely to none" and is not claimed. Microservices
is worked-_within_, not a distributed-messaging/event-driven claim. Azure is
present via Azure SQL Server. When unsure, undersell.

**"How I build" (Landing philosophy surface)**:
The home-page section that carries the engineering philosophy in voice — the
hook. It gestures at the stack but is not a list. Distinct from the About
evidence surface.
_Avoid_: tech stack section, skills section

**About evidence surface**:
The single canonical place where the concrete, current stack is stated plainly
for a scanning recruiter. Proof, not pitch.
_Avoid_: skills matrix
