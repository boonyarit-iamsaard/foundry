# AGENTS.md

Guidance for AI agents working in this repository. `CLAUDE.md` is a symlink to
this file.

## Project

`boonyarit.me` — the personal portfolio of Boonyarit Iamsa-ard, a full-stack
developer based in Bangkok. A content-first site (hero, projects, articles,
about, contact) built as a **brand** surface: the design is part of the product.

**Stack:** Next.js 16 (App Router, Turbopack) · React 19 · TypeScript ·
Tailwind CSS v4 · shadcn/ui (new-york) · Velite (MDX content) · forms via
next-safe-action, react-hook-form, and Zod · email via Resend/Nodemailer ·
theming via next-themes (light/dark).

Requires Node 24 and pnpm 11 (`nvm use`). Create `.env` from `.env.example`
before type-checking or building.

## Commands

```sh
pnpm dev                 # next dev --turbopack
pnpm build               # velite build --clean && next build
pnpm run static-analysis # prettier --check . && lint:check && type-check
pnpm run format          # prettier --write .
pnpm run lint            # eslint --fix --max-warnings=0
pnpm run type-check      # next typegen && tsc --noEmit
pnpm run lint:markdown   # markdownlint over .md/.mdx (local-only, not in CI)
pnpm dev:email           # preview React Email templates on :4000
```

Content types are generated by Velite; run `pnpm exec velite build --clean`
before type-checking directly. A Husky + lint-staged pre-commit hook runs
Prettier/ESLint on staged files. CI runs `static-analysis` only;
`lint:markdown` is a local check and intentionally does not gate deployment.

## Structure & conventions

- `src/app/` — Next.js App Router routes (`/`, `/about`, `/projects`,
  `/articles`, plus `[slug]` detail pages).
- `src/common/` — shared components (`ui/` = shadcn primitives), helpers,
  validators, definitions.
- `src/core/` — app config, env config, fonts, global styles, assets.
- `src/features/` — feature modules (`landing`, `articles`, `projects`,
  `emails`), each with its own `components/`, `actions/`, `hooks/`, `validators/`.
- `content/` — MDX articles, projects, and `about.mdx`, compiled by Velite.
- Path aliases (see `components.json` / `tsconfig.json`): `@/common/components`,
  `@/common/helpers`, `@/core/...`, etc. Use `cn` from `@/common/helpers/cn`.
- Imports are auto-sorted by `@ianvs/prettier-plugin-sort-imports`; let Prettier
  own formatting rather than hand-ordering.

## Design Context

Two root files define the design intent — **read them before any UI work**:

- **`PRODUCT.md`** — strategic: register (`brand`), audiences, product purpose,
  brand personality, anti-references, and design principles.
- **`DESIGN.md`** — visual system: OKLCH tokens, typography, components, and
  Do's/Don'ts. North Star: **"The Engineer's Field Notes."**

Doctrine in brief: **one warm terracotta** on cool neutral paper (≤10% of any
screen) · **mono (JetBrains Mono) for metadata only**, never headings/body/CTAs ·
**flat at rest**, elevation only on hover/focus · editorial restraint, WCAG AA.
`.impeccable/` holds the design sidecar and live-mode config (impeccable skill).

## Agent skills

### Issue tracker

Issues and PRDs live as local markdown under `.scratch/<feature>/`; triage state
is a `Status:` line in each issue file (no external tracker, no PR triage
surface). See `docs/agents/issue-tracker.md`.

### Triage labels

The five canonical triage roles use their default strings: `needs-triage`,
`needs-info`, `ready-for-agent`, `ready-for-human`, `wontfix`. See
`docs/agents/triage-labels.md`.

### Domain docs

Single-context: one `CONTEXT.md` + `docs/adr/` at the repo root. See
`docs/agents/domain.md`.
